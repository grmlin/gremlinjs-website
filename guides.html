Introduction

GremlinJS allows you to organize your Javascripts in components called Gremlins.

Every dom element may use one or more of these components. GremlinJS searches and finds all these elements (no matter if they are part of the dom from the beginning or added later)and creates Gremlin instances of the Gremlin definitions you provided.

Gremlins are found by a special data-attribute: data-gremlin. This happens automatically whenever a dom element with this attribute is or becomes a child node of the dom.
Yepp, GremlinJS observes the dom for changes!
In websites build the classical, jQuery-style way, you normally end up with a huge list of dom-queries initializing all your javascript-enhancements. GremlinJS uses a single query for all your elements.

Normally the components you'll write are isolated from each other and function independently. To use a Gremlin in another project, all you'll do is copy a file and add some HTML markup.

Include GremlinJS

Download GremlinJS and include it in your HTML. From now on all gremlin components in the site will be found and instantiated. 

<!DOCTYPE html>
<html>
  <head>
  </head>
  <body>
    ...
    <script src="js/gremlinjs.min.js"></script>
  </body>
</html>

If you have to support an old browser like IE7, be sure to include a JSON Library like json2.js before including GremlinJS.
GremlinJS uses the javascript JSON API to parse JSON strings in custom dom element data-attributes. (see Gremlin Configuration)
