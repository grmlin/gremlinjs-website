Introduction

GremlinJS allows you to organize your Javascripts in components called Gremlins.

Every dom element may use one or more of these components. GremlinJS watches your site looking for these elements and creates Gremlin instances of Gremlin definitions (aka classes) you provided.

Gremlins are found with a special data-attribute: data-gremlin. This happens automagically whenever a dom element with this attribute is or becomes a child node of the dom.

Yepp, GremlinJS observes the dom for you.

Components you'll write with GremlinJS are isolated from each other and function independently. To use a Gremlin in another project, all you'll do is copy a file and add some HTML markup.
The code will be cleaner and easier to understand: whenever you see an element with a data-gremlin attribute, you will always and instantly know what javascript is responsible for it's behaviour.

Include GremlinJS

Download GremlinJS and include it in your HTML. From now on all gremlin components in the site will be found and instantiated.

You don't have to start or initialize it, you don't have to wait for anything, include it and you're done. 

<!DOCTYPE html>
<html>
  <head>
  </head>
  <body>
    <script src="js/gremlinjs.min.js"></script>
  </body>
</html>

If you have to support an old browser (eg IE7), be sure to include a JSON Library like json2.js before including GremlinJS.
GremlinJS uses the javascript JSON API to parse JSON strings in custom dom element data-attributes. (see Gremlin Configuration)

The first Gremlin

Gremlins are build by writing a javascript definition and added to dom elements via a custom data-attribute, data-gremlin.

Gremlin definitions

Gremlin definitions are Javascript snippets describing the gremlin. You will create constructor functions which are inherited from an abstract gremlin class behind the scenes by GremlinJS.

If GremlinJS finds a suitable dom element, one that provides the data-gremlin attribute, a new instance of the constructor function, the gremlin class, will be created. This instance does, beside the members and methods you defined, provide some useful properties you may need later.

GremlinJS uses constructor functions, the protoype chain and the new keyword to create gremlin instances internally. So no matter how many gremlins of a certain type are present in the document and instantiated, they'll use the beauty and performance of the prototype.

Creating a definition

To make the developer's life easier, GremlinJS provides a helper method to add gremlin definitions.

GremlinJS.define()

Creates a gremlin definition that will later be used to activate gremlin elements in the document.
This will be a native javascript constructor function instantiated with new using the prototype and static members.

GremlinJS.define(name, constructor, instanceMembers, staticMembers)
  • name

    The unique name of the gremlin.

  • constructor

    The constructor function being called every time a gremlin with the name defined above was found in the dom.
    this inside the constructor refers to the gremlin instance.

  • instanceMembers

    All instance members of this class as an object literal. Will be mixed into the prototype of the gremlin (the constructor function).

  • staticMembers

    All static members of this class as an object literal.
    To access static members from inside gremlin instances, <gremlinInstance>.klass references the original constructor function.
